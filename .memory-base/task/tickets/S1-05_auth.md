Код: S1-05 — Auth (регистрация/логин/approve)

Цель
- Регистрация PENDING, логин, approve админом, роли ADMIN/USER.

Описание
- Реализовать аутентификацию по email+паролю с выдачей JWT (в httpOnly Secure cookie). Новые пользователи получают статус PENDING и видимы администратору для подтверждения. Доступ к персональным данным ограничен ролью/статусом.

Шаги
- Эндпоинты `/auth/register`, `/auth/login`, `/auth/approve`.
- JWT cookie Secure HttpOnly (за NPM).

Тестирование (обязательно)
- Позитивные/негативные сценарии регистрации/логина, блокировка PENDING.
- Проверка RBAC на защищённых эндпоинтах, корректность куки за NPM.

AC: RBAC базовый, негативные кейсы покрыты. Зависимости: S1-04.
