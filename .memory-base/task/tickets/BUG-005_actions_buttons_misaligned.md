ID: BUG-005
Title: Несимметричное выравнивание кнопок действий — ломается визуальный стиль списка/детальной карточки
Type: bug
Priority: P2
Status: Open
Owner: frontend
Created: 2025-11-11

Кратко
— Блок «Действия» рендерит кнопки с нарушенным выравниванием и несогласованными размерами: элементы «Открыть/Удалить» в списках и «Скачать/Редактировать метрики/Просмотр/Удалить» в карточке отображаются как разъехавшиеся столбцы/ряды, что ломает визуальную композицию.

Контекст/наблюдения
— На странице со списком сущностей (например, отчётов/участников) вертикальные интервалы и ширины кнопок различаются, кнопки «Открыть» и «Удалить» складываются в две строки с разными отступами, выглядит «ступеньками».  
— На детальной странице (карточке) кнопки действий имеют разные высоты/паддинги; красная кнопка удаления визуально «выпадает» из ряда.  
— Предположительно, конфликт глобальных стилей с Element Plus (`ElButton`), где переопределены `padding`, `line-height`, `display` или используется смешанная разметка (`inline-block` vs `flex`).

Окружение
— macOS 15 (Darwin 25.0.0), Chrome (актуальный), локальная среда.  
— Воспроизводится на текущей ветке.

Шаги воспроизведения
1) Открыть список сущностей с колонкой «Действия» (например, отчёты).  
2) Проскроллить и сравнить карточки — заметны разные отступы/высоты у «Открыть» и «Удалить».  
3) Открыть детальную страницу отчёта/участника.  
4) В блоке «Действия» сравнить «Скачать», «Редактировать метрики», «Просмотр» и кнопку удаления — элементы не выровнены по вертикали и имеют разные размеры.

Фактический результат
— Кнопки в блоке «Действия» выровнены неровно, с различными высотами и паддингами; нарушена сетка и визуальная целостность.

Ожидаемый результат
— Единое выравнивание по базовой линии/центру для всех кнопок; одинаковая высота, паддинги и межкнопочные отступы согласно дизайну. В списках — аккуратная горизонтальная группа или корректная вертикальная колонка без «ступенек».

Предполагаемая причина
— Смешение `display` режимов и/или кастомных токенов кнопок: глобальные CSS-правила в `frontend/public/assets/theme-tokens.css` или локальные стили компонентов переопределяют `--el-button-*`, `line-height`, `padding`, `gap`.  
— Неконсистентная разметка: разные контейнеры используют `flex`/`grid`/`inline-block`, отсутствуют единые классы группы действий.

Затронутые компоненты/места
— Vue-страницы со списками (таблицы/карточки) и детальные карточки: файлы в `frontend/src/views/*` и/или `frontend/src/components/*`, где рендерится колонка «Действия».  
— Глобальные стили и токены: `frontend/public/assets/theme-tokens.css`.  
— Возможные локальные стили `.actions`, `.action-buttons`, обёртки `ElButton`.

Критерии приёмки
— Все кнопки действий в списках и детальных карточках:  
  • визуально выровнены по центру по вертикали в одном ряду;  
  • имеют унифицированную высоту и отступы;  
  • межкнопочные расстояния постоянны;  
  • мобильная/узкая ширина — перенос в следующую строку с сохранением равномерных интервалов.  
— Проверка в последних Chrome/Firefox.  
— E2E критический путь проходит без адаптаций.

Технические заметки/направления фикса
— Ввести единый контейнер группы действий (например, класс `.actions-row`) с `display: flex; align-items: center; gap: 8px; flex-wrap: wrap;`.  
— Запретить локальные переопределения `line-height`/`padding` у отдельных `ElButton`, полагаться на токены Element Plus.  
— В `theme-tokens.css` выровнять токены кнопок (primary/success/danger) по высоте/паддингам.  
— Для колонок таблиц задать фиксированный вертикальный ритм (min-height, выравнивание содержимого ячейки по центру).

Связанные артефакты/скриншоты
— Скриншоты приложены в пользовательском сообщении к задаче (листинг с «Открыть/Удалить», карточка с «Скачать/Редактировать/Просмотр/Удалить»).

Risks
— Возможное влияние на все кнопки Element Plus после выравнивания токенов; потребуется регресс основных форм и тулбаров.

Next actions
— Прототип единой разметки группы действий и подбор токенов/отступов; визуальная проверка списка и карточки.
