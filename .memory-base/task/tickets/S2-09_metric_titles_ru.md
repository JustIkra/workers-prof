ID: S2-09
Title: Названия метрик в интерфейсе отображаются на английском
Type: bug
Priority: P2
Status: Open
Owner: frontend
Created: 2025-11-13

Кратко
— Колонка «Название метрики» в карточке участника отображает английские строки из `MetricDef.name` (например, `Abstractness`, `Achievement Motivation`) вместо ожидаемых русских названий.

Контекст
— В `frontend/src/views/ParticipantDetailView.vue` таблица «Актуальные метрики участника» использует `getMetricName`, который берет `metricDefs.value` из `/api/metric-defs`.
— `metricsApi.listMetricDefs` возвращает `MetricDefResponse`, где поле `name` заполнено данными из сидов/импорта AI и содержит английские значения.
— В `config/app/metric-mapping.yaml` и в исходных DOCX после распознавания (ИИ) уже присутствуют русские ярлыки, на которые ориентируются операторы.

Проблема
1. Пользователь видит англоязычные заголовки при полностью русскоязычном интерфейсе, что воспринимается как ошибка локализации.
2. Операторам сложно сопоставить значения с исходным отчётом: названия в UI не совпадают с теми, что выдал ИИ/таблица.
3. У нас нет единого источника русских названий: фронтенд вынужден мапить по коду, а `MetricDef` хранит только английский `name`.

Ожидаемое поведение
— Колонка «Название метрики» всегда показывает русские заголовки, совпадающие с первоначальными ярлыками после ИИ (например «Анализ и планирование», «Тревожность»).
— При отсутствии русского значения допустимо показывать код метрики, но не англоязычное название.

Зона изменений
- `api-gateway`:
  - Добавить хранение и отдачу русских названий (поле `display_name_ru` / `name_ru` в `MetricDef` либо выдавать данные через `MetricMappingService`).
  - Обновить схемы и сиды, подставив русские строки из исходных ярлыков ИИ (`config/app/metric-mapping.yaml`).
- `frontend/src/views/ParticipantDetailView.vue`:
  - Переключить `getMetricName` на использование нового поля из API, убрать зависимость от английского `name`.
  - Проверить, что редактор метрик и другие компоненты используют тот же источник.

Тестирование
- Backend: дополнить `pytest`-тесты для `/api/metric-defs`, проверяющие наличие русских названий.
- Frontend: добавить единичный тест для `ParticipantDetailView`/store, убеждающийся, что таблица отображает русские ярлыки.
- E2E: Прогнать сценарий S2-08 (Playwright) и заскриншотить, что «Название метрики» отображается на русском.

Критерии приёмки
- `/api/metric-defs` возвращает русские названия, совпадающие с `metric-mapping.yaml`.
- Интерфейс карточки участника показывает русские заголовки для всех активных метрик.
- При отсутствии русской строки UI показывает код и пишет предупреждение в консоль.
- Документация по сидированию/обновлению метрик обновлена с указанием, что название берётся из первоначальных ярлыков после ИИ.
