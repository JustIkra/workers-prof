# UX-05: Обновление интерфейса раздела «Отчёты»

Статус: Planned  
Приоритет: Medium  
Затронутые части: frontend (Vue 3, Element Plus), UX-дизайн, e2e-тесты  
Ответственный: Web UI

## Контекст
Текущий список отчётов представляет собой таблицу с минимальным форматированием. Пользователи жалуются на сложность визуального восприятия и отсутствие подсказок: нажатие по названию непонятно ведёт ли к скачиванию, неочевидно какие действия доступны в зависимости от статуса, сложно ориентироваться при большом количестве строк и на мобильных устройствах.

Скриншот из продакшена (11.11.2025) приложен в обращении — кнопки действий визуально перегружены, статусы сливаются между строками, отсутствует явная группировка и сортировка по дате.

## Цель
Переработать фронтенд раздела «Отчёты», чтобы:
- улучшить визуальную иерархию и читаемость;
- упростить сценарии «скачать», «просмотреть», «редактировать метрики»;
- поддержать сортировку и фильтрацию;
- обеспечить корректный вид на экранах ≤1024px.

## Требования к UX/UI
- **Список**: использовать карточки или расширенную таблицу с чётким разделением строк, отступами и фоновыми подложками. Вывести колонку «Тип отчёта», «Статус», «Дата загрузки», «Автор (если доступно)».
- **Действия**: сгруппировать кнопки действий в один блок с dropdown или split-button. На десктопе отображать основные действия («Просмотр», «Редактировать метрики») как первичные, дополнительные («Скачать», «Удалить») — в меню. На мобильных устройствах — иконки + текст.
- **Статусы**: унифицировать бейджи (цветовая схема Element Plus), добавить иконки состояния (извлекается / извлечено / ошибка). Предусмотреть skeleton/loader для строк со статусом «в обработке».
- **Навигация**: клик по названию/типу открывает просмотр отчёта; кнопка скачивания — отдельное действие, недоступное по умолчанию.
- **Пустые состояния**: добавить иллюстрацию и CTA «Загрузить отчёт».
- **Фильтры и сортировка**: предусмотреть фильтр по статусу (select) и соритровку по дате загрузки (последние сверху). Хранить состояние фильтров в URL query для шаринга.
- **Адаптивность**: на узких экранах перестраивать список в карточки; действия — в нижней части карточки.
- **Доступность**: добавить aria-лейблы для кнопок, обеспечить контрастность (WCAG AA).

## Технические заметки
- Компонент расположен в `frontend/src/views/ParticipantsView.vue`; вынести список отчётов в отдельный компонент (например, `ReportList.vue`) с пропсами данных.
- Реализовать Pinia-стор или Composition API-композит для фильтрации и сортировки, чтобы не дублировать логику.
- Использовать существующие API `/api/reports`. При необходимости запросить дополнительное поле «author_name».
- Добавить Vitest-снепшоты и тесты на клик по основным действиям.
- Обновить Playwright-сценарий (`e2e/ui-calculate-score.spec.js`), чтобы учитывать новый порядок действий и селекторы.

## Критерии приёмки
- Пользователь видит переработанный список с понятной иерархией, статусы различимы визуально.
- Основные действия доступны за ≤2 клика, подтверждения для удаления присутствуют.
- Фильтр по статусу и сортировка по дате работают и сохраняются при обновлении страницы.
- UI корректно отображается на ширине 375px и 1440px (ручная проверка).
- Vitest и Playwright тесты обновлены и проходят.

## Оценка
- Дизайн: 4–6 ч (прототип + review)
- Реализация: 8–10 ч
- Тесты и регрессия: 3–4 ч
