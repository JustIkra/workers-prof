Title: Обновить текст рекомендаций без ссылок на курсы
Type: enhancement
Priority: P2
Status: Planned
Owner: content+frontend
Created: 2025-11-12

Кратко
— В блоке «Рекомендации» итогового отчёта нужно заменить формулировки: вместо ссылок на конкретные курсы предоставить описания, какие навыки развивать и какие форматы обучения могут помочь, без прямых URL.

Контекст
— Текущая версия UI отображает список рекомендаций вида «Курс по дизайну-мышлению (перейти к курсу)» со ссылкой на внешний ресурс.
— Пользователи жалуются, что ссылки быстро устаревают или ведут на платные курсы, а им важнее понять, какие компетенции развивать и какие шаги предпринять.
— Скриншот от 12.11.2025 показывает пять элементов списка, каждый содержит ссылку «перейти к курсу».

Задача
1. Backend / генерация рекомендаций:
   - Обновить схему и шаблон генерации, чтобы в JSON/HTML итогового отчёта возвращались описательные рекомендации: «Развивайте X, полезны Y форматы (мастер-классы, менторинг, практикумы)», без полей `link` или `cta_url`.
   - Убедиться, что пайплайн ИИ корректно формирует текст в нужной структуре (например, `focus`, `how_to_develop`, `resources_hint`) и что ручные черновики/шаблоны тоже обновлены.
2. Frontend:
   - Обновить отображение блока `FinalReportRecommendations`, удалив вывод ссылок и CTA.
   - Адаптировать компонент под новую структуру данных: заголовок рекомендации, текст с советами по развитию, список форматов (если есть).
3. Контент и UX:
   - Согласовать тональность текста (мотивационный, без прямых призывов к покупке).
   - Добавить подсказку «Подберите удобный формат: воркшоп, наставник, профессиональные сообщества» вместо ссылок.

Требования к данным
- JSON итогового отчёта содержит массив объектов с полями `title`, `skill_focus`, `development_advice`, `recommended_formats` (список строк). Ссылочные поля отсутствуют.
- HTML-версия отображает текстовые советы и списки без гиперссылок.
- При отсутствии `recommended_formats` блок отображает только `development_advice`.

Тестирование
- Backend: pytest на генерацию итогового отчёта — убедиться, что `link` не присутствует, а новые поля заполнены.
- Frontend: Vitest компонент `FinalReportRecommendations` — отображение текста и маркированного списка без ссылок.
- Playwright: сценарий просмотра отчёта проверяет, что в DOM нет `<a>` внутри блока рекомендаций.

Критерии приёмки
- В отчёте рекомендации описывают навыки и шаги развития, не содержат ссылок.
- Пользователь видит понятные подсказки по форматам развития (мастер-классы, наставничество, практические задания и т.д.).
- Старые данные с `link` либо мигрированы, либо корректно отображаются без кликабельных ссылок (fallback).
- Документация (`.memory-base/Tech details/infrastructure/prompt-gemini-recommendations.md`) обновлена под новую структуру.
