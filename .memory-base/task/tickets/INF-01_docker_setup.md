Код: INF-01 — Настройка запуска проекта в Docker

Цель
- Обеспечить корректный запуск всего стека приложений через `docker-compose up` с автоматической инициализацией БД, миграциями и проверкой работоспособности всех сервисов.

Описание
- Проект должен запускаться одной командой `docker-compose up -d` с автоматической настройкой окружения. Необходимо проверить и при необходимости доработать:
  - Корректность сборки образов (app, worker)
  - Инициализацию БД (миграции Alembic)
  - Health checks для всех сервисов
  - Правильную работу entrypoint скриптов
  - Доступность API после запуска
  - Работу Celery worker
  - Настройку переменных окружения через .env

Шаги
- Проверить сборку образов: `docker-compose build`
- Проверить запуск стека: `docker-compose up -d`
- Убедиться, что миграции применяются автоматически при старте app контейнера
- Проверить health checks для postgres, redis, rabbitmq
- Проверить доступность API: `curl http://localhost:9187/api/healthz`
- Проверить работу Celery worker (логи, обработка задач)
- Убедиться, что .env.example содержит все необходимые переменные
- Проверить, что volumes корректно монтируются (storage для файлов)
- Документировать процесс запуска в README или CLAUDE.md

Тестирование (обязательно)
- `docker-compose up -d` запускает все сервисы без ошибок
- API доступен на порту 9187
- Health endpoint возвращает 200 OK
- Миграции применены (проверка через `docker-compose exec app alembic current`)
- Celery worker обрабатывает задачи (можно проверить через логи)
- Все зависимости (postgres, redis, rabbitmq) доступны внутри сети Docker

AC: проект запускается одной командой; все сервисы здоровы; API доступен; миграции применены; worker работает. Зависимости: нет.
