Код: S1-01 — Единственный `.env` и загрузка конфигурации

Цель
- Использовать один корневой `.env` как основной источник конфигурации (порт 9187, DSN, VPN, Gemini и пр.).

Описание
- Один корневой `.env` определяет все переменные среды для приложения и Compose. В нём задаются порт `9187`, прокси‑заголовки за NPM, DSN БД/кэша/очереди, параметры файлового хранилища, флаги VPN (WireGuard) и пул ключей Gemini. Реальные секреты не хранятся в VCS.

Шаги
- Загрузка `.env` на старте (Pydantic Settings).
- Запрет дублирующих `.env.*` (в коде/доках).
- Документация о политике одного `.env`.

Тестирование (обязательно)
- Проверить, что значения из `.env` применяются (порт, DSN) и отсутствуют обращения к иным `.env.*`.
- Неверные/пустые значения приводят к предсказуемым ошибкам конфигурации.

Acceptance Criteria
- Приложение читает переменные только из корневого `.env`.
- Документация обновлена; лишние примеры удалены.

Зависимости: нет.
