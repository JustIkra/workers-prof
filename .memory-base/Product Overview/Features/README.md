Ключевые фичи и критерии готовности

1) Управление участниками
- Добавление/редактирование участника (ФИО, дата рождения, внешний ID)
- Поиск/фильтр по ФИО/ID
- Готовность: CRUD работает, валидации, аудит

2) Загрузка отчётов (.docx)
- Привязка к участнику, тип отчёта (1/2/3)
- Хранение файла локально (volume), выдача ссылки на скачивание
- Готовность: файл доступен, размер и MIME сохраняются

3) Извлечение числовых метрик из таблиц‑картинок
- Gemini Vision (по умолчанию) для извлечения чисел; локальный OCR (PaddleOCR + PP-Structure) — fallback
- Нормализация в словарь метрик по справочнику MetricDef
- Готовность: ≥ 95% корректных полей на контрольных примерах; ручная валидация в UI

4) Весовые таблицы по профдеятельности
- Загрузка администратором, версионирование, активация одной версии
- Валидация суммы весов = 1.0
- Готовность: активная таблица выбирается автоматически при расчёте

5) Расчёт коэффициента пригодности
- Формула на основе весов и метрик; результат в %
- Генерация сильных сторон/зон развития/рекомендаций
- Готовность: повторяемость результата на тестовых данных

6) Роли и доступ
- Роли: ADMIN, USER; регистрация с подтверждением админа
- Доступ к персональным данным только после одобрения
- Готовность: защита эндпоинтов, корректные JWT

7) Аудит и журналирование
- Логи API/воркеров; отслеживание версий весовых таблиц
- Готовность: базовые дашборды/метрики (опционально)
