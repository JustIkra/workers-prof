# User Flow — Цифровая модель универсальных компетенций (ЦМУК)

## Контекст
- Продукт доступен как веб-приложение «Цифровая модель универсальных компетенций».
- Пользовательские роли: новый посетитель (регистрация) или существующий пользователь (вход по логину/паролю).

## Основной поток
1. Пользователь заходит на сайт и регистрирует личный кабинет либо авторизуется в существующем.
2. После входа открывается рабочая область с вкладками «Настройки», «Участники тестирования» и др.; ключевой сценарий проходит через раздел участников.
3. В разделе участников создаётся карточка человека (ФИО, дата рождения, внешний идентификатор) и открывается его личная страница.
4. На странице участника можно загрузить и просмотреть **три** отчёта в формате `.docx`. Все отчёты одинаково структурированы, но содержат разные наборы переменных/выводов.
5. Каждый отчёт включает таблицы, встроенные как изображения (`word/media/*`). Система извлекает изображения, прогоняет через OCR/LLM и превращает значения в набор числовых характеристик по справочнику `MetricDef`.
6. Пользователь выбирает профессиональную деятельность (например, «Проведение совещаний»). Для каждой деятельности существует версия весовой таблицы: сумма весов = 1.0, а набор метрик соответствует выбранной области.
7. Числовые показатели участника подставляются в формулу `Σ(value × weight) × 10`, что даёт коэффициент профессиональной пригодности в процентах.
8. Результат фиксируется в карточке участника и формирует итоговый отчёт: сильные стороны (3–5), зоны развития (3–5) и рекомендации по обучению/тренингам.

## Входные артефакты
- Примеры реальных отчётов находятся в этой же папке:
  - `Batura_A.A._Biznes-Profil_Biznes-otchyot_1718107.docx`
  - `Batura_A.A._Biznes-Profil_Otchyot_dlya_respondenta_1718107.docx`
  - `Batura_A.A._Biznes-Profil_Otchyot_po_kompetentsiyam_1718107.docx`
  - `Пример_расчет_Проведение_совещаний.docx`
- Эти файлы используются для проверки структуры DOCX, таблиц-изображений и эталонных значений метрик.

## Выходы
- Коэффициент пригодности в диапазоне 0–100%.
- Итоговый отчёт с перечислением сильных сторон, зон развития и рекомендаций (обучение, тренинги, программы развития).
- История расчётов и версия активной весовой таблицы фиксируются в базе данных для аудита.
