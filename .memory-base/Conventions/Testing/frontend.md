Frontend: тестирование Vue 3

Инструменты
- Unit: Vitest + Vue Test Utils.
- E2E: Playwright.

Нейминг и каталоги
- `src/components/__tests__/...` для unit близко к компоненту.
- E2E: `e2e/` в корне фронтенда.

Unit‑правила
- Тестировать логику компонентов, стора (Pinia), форматирование/валидацию.
- Мокать HTTP сервисы (Axios) и маршрутизацию.
- Использовать `data-testid` для стабильных селекторов.

E2E‑правила
- Поднимать фронт и API (docker compose) или использовать `devserver` в Playwright config.
- Хранить креды тест‑юзеров в `.env.e2e`.
- Записывать видео/скриншоты на падениях; отчёты хранить до 7 дней.

Критичные сценарии (минимум)
- Логин/логаут.
- Админ: approve пользователя, загрузка и активация весов.
- Пользователь: создать участника, загрузить .docx, ввод метрик, расчёт, скачать DOCX.

Принципы стабильности
- Явные ожидания по видимости/состоянию (expect(locator).toBeVisible()).
- `data-testid` вместо CSS/XPath.
- Избегать кликов «по координатам».
