E2E матрица сценариев (Playwright)

Сценарий 1: Login/Admin approve
- Шаги: регистрация user → логин админом → approve → логин юзером.
- Проверки: статус/уведомления; роль и доступы.

Сценарий 2: Управление весами
- Шаги: админ загружает JSON весов → активирует.
- Проверки: сумма=1.0, активная версия отображается; попытка второй активации блокируется.

Сценарий 3: Участники и отчёты
- Шаги: создать участника → загрузить .docx → скачать исходник.
- Проверки: файл загружен, ссылка работает, запись видна в списке.

Сценарий 4: Метрики и расчёт
- Шаги: открыть отчёт → ввести/подтвердить метрики (временно) → выбрать профобласть → «Рассчитать» → скачать DOCX.
- Проверки: корректный score, наличие ключевых секций в DOCX.

Сценарий 5 (после извлечения Vision/OCR): Извлечение → расчёт
- Шаги: нажать «Извлечь из отчёта» → дождаться статуса → рассчитать → рекомендации.
- Проверки: значения отображаются с подсветкой; рекомендаций ≤ 5; JSON валиден.

Сценарий 9: Финальный отчёт (JSON)
- Шаги: открыть карточку участника → убедиться, что есть расчёт → нажать «Просмотреть JSON» → открыть новую вкладку с JSON.
- Проверки: структура совпадает с `FinalReportResponse`; ключевые поля заполнены; без ошибок в консоли.

Сценарий 10: Финальный отчёт (HTML)
- Шаги: открыть карточку участника → нажать «Скачать HTML» → сохранить файл → открыть в браузере.
- Проверки: HTML рендерится; присутствуют основные секции (заголовок, score, списки); без артефактов разметки.
