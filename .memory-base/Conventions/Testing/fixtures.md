Фикстуры и тестовые данные

Принципы
- Только синтетические данные; без ПДн.
- Документы .docx в `tests/fixtures/documents/` с уменьшенными изображениями.
- Все артефакты (временные файлы, скриншоты) складывать в tmp и чистить.

Типовые фикстуры
- `user_admin`, `user_active` — создают пользователей через репозитории/фабрики.
- `participant_factory` — создает участника с уникальными полями.
- `weight_table_active` — активные веса для выбранной профобласти.
- `report_with_docx` — загруженный отчёт с привязанным file_ref (temp файл).
- `vision_stub` — monkeypatch Vision клиент, возвращает числа по коду метрики.
- `gemini_stub` — возвращает фиксированный JSON рекомендаций.

Фабрики
- Предпочтительно простые фабрики/билдеры (dataclass/pydantic), без тяжёлых зависимостей.
- Генераторы значений — детерминированные (seeded faker).

Управление временем
- `freezegun` фиксирует `now()`; расчёты и метки времени сравниваются строго.
