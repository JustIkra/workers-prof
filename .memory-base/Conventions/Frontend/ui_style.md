UI стиль: «офисный» для гос-сферы

Цель
- Задать строгий, «офисный» стиль интерфейса: функциональный, нейтральный, с высокой читаемостью и доступностью. Подходит для рабочих мест в госсекторе (часто Windows, невысокие DPI, требования к контрасту и печати).

Принципы
- Сдержанность: минимум декоративных эффектов, отсутствие «игровых» анимаций.
- Консервативные шаблоны: форма слева → таблица справа; боковое меню; кнопки «Сохранить/Отмена» внизу справа.
- Предсказуемость: всегда одинаковое расположение действий, одинаковые названия.
- Доступность: контраст WCAG AA, работа с клавиатурой, явные фокусы.
- Локализация: ru-RU (даты dd.mm.yyyy, 24‑часовой формат, десятичная запятая).

Типографика
- Базовый кегль: 14 px; line-height: 1.5.
- Семейство шрифтов (системный стек, без внешних загрузок):
  - Windows: "Segoe UI", Tahoma, Arial
  - macOS: -apple-system, "SF Pro Text"
  - Linux: system-ui, "Ubuntu", "Cantarell"
  - Стек: `font-family: Segoe UI, Tahoma, Arial, -apple-system, system-ui, sans-serif;`
- Заголовки: H1=20 px, H2=18 px, H3=16 px; полужирный, без лишних цветов.

Цветовая палитра (тон «офисный», близко к примерам отчётов)
- Primary (бирюзово‑синий): `#00798D`
- Primary Dark: `#025E6D`
- Primary Light: `#E0F3F6`
- Info/Link: `#0B5CAD`
- Success: `#2E7D32`
- Warning: `#B26A00`
- Danger: `#C62828`
- Gray‑900: `#1F2937`, Gray‑700: `#374151`, Gray‑500: `#6B7280`, Gray‑200: `#E5E7EB`, Gray‑50: `#F9FAFB`
- Focus outline: `#0B5CAD` (2 px, сплошной)

Компоновка
- Хедер (64 px) с логотипом/названием; слева вертикальное меню (ширина 240 px).
- Контент: max‑width 1280 px; поля 24 px; сетка 12 колонок.
- Печать: опционально; при необходимости — отдельный print‑CSS (белый фон, чёрный текст, без навигации).

Компоненты (поведение и внешний вид)
- Кнопки
  - Primary: сплошная заливка Primary; hover: затемнение 8%; disabled: 40% opacity.
  - Secondary: светлая заливка Gray‑50, бордер Gray‑200; текст Gray‑900.
  - Link: цвет Info/Link; подчёркивание только по hover/focus.
- Формы
  - Лейблы слева, поля справа; обязательные поля со звёздочкой *. Подсказка/ошибка под полем малым серым/красным.
  - Маски: дата dd.mm.yyyy; числа — десятичная запятая; проценты — «%» справа.
  - Группы полей разделять секциями с заголовками H3.
- Таблицы
  - Высота строки 44 px; шрифт 14 px; зебра строки (Gray‑50), фиксированный заголовок, сортировка по столбцам.
  - В строке — явные текстовые действия: «Открыть», «Загрузить», «Удалить»; иконки вторичны.
  - Пагинация внизу справа; размеры страницы 10/20/50.
- Модальные окна
  - Ширина 560–720 px; заголовок H2; крупные кнопки «Сохранить/Отмена»; закрытие Esc + крестик.
- Загрузка файлов
  - Dropzone с бордером Gray‑200; подсказка «Поддерживаются .docx»; прогресс‑бар; ошибки в явном алерте.
- Уведомления
  - Ненавязчивые Toast’ы вверху справа; важные ошибки — Alert в контенте.
- Иконки
  - Лаконичные, монохромные; набор Element Plus Icons или Material Outlined; размер 16/20 px.

Библиотека UI
- По умолчанию используем Element Plus (предпочтительно для «офисного» стиля и доступности).
- Альтернатива — Naive UI; токены темы совместимы.

Доступность
- Цветовой контраст не ниже AA; проверка в CI (stylelint‑a11y опц.).
- Навигация табом по всем интерактивным элементам; видимый фокус.
- ARIA для форм/алертов; `aria-describedby` для ошибок ввода.

Числа/датавременные форматы
- ru-RU: `01.02.2025`, время `14:05`, разделитель тысяч — пробел, десятичный — запятая.
- Ввод чисел — только цифровая клавиатура; нормализация на бекенде (запятая→точка).

Темизация через CSS‑переменные (совместимо с Element Plus/Naive UI)
- Базовые токены задаются в `:root` (см. `theme-tokens.css`).
- Для Element Plus сопоставить:
  - `--el-color-primary: var(--color-primary)`
  - `--el-font-size-base: 14px`
  - `--el-border-radius-base: 4px`
  - `--el-color-success/warning/danger/info` — согласно палитре
- Для Naive UI — задать `themeOverrides` с той же палитрой.

Микрокопирайтинг
- Деловой, нейтральный тон: «Загрузить», «Сохранить», «Отменить», «Рассчитать», «Сгенерировать рекомендации».
- Текст ошибок конкретный: «Файл должен быть .docx и не больше 20 МБ».

Переиспользуемые макеты
- Карточка участника: секции «Паспорт», «Отчёты», «Результаты»; справа — действия.
- Экран весов: список версий слева, активная версия и строки весов справа.
